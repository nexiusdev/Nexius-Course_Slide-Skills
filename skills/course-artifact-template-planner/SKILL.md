# Course Artifact Template Planner

## Purpose

Use this skill for artifact-producing courses. Its job is to identify every non-slide asset learners need to complete the course successfully.

Do not build the full PowerPoint here. Plan the handouts, canvases, worksheets, checklists, rubrics, and final deliverable templates that support the learning experience.

## Required Inputs

Start from:

- course brief or approval document,
- learning experience blueprint,
- slide inventory if available,
- expected participant deliverables,
- final management or business outputs,
- delivery format: printed handout, Word, Excel, Google Docs, PowerPoint, PDF, or in-deck canvas.

## Artifact Planning Workflow

1. Extract every promised course output.
2. Separate outputs into:
   - learner worksheet,
   - team canvas,
   - checklist,
   - rubric,
   - reference guide,
   - sample/example,
   - executive final deliverable,
   - facilitator-only guide.
3. For each artifact, define:
   - purpose,
   - owner or user,
   - when used in the course,
   - required fields or sections,
   - input sources,
   - completion criteria,
   - review method,
   - final format.
4. Add scaffolding:
   - instructions,
   - examples,
   - field descriptions,
   - quality checklist,
   - common mistakes,
   - debrief prompts.
5. Map artifacts to slides and activities.
6. Flag missing artifacts where a slide asks learners to do work but no template exists.
7. Flag overbuilt artifacts where a simple checklist or in-slide canvas is enough.

## Artifact Types

Use these common formats:

- worksheet for individual reflection or drafting,
- team canvas for collaborative activity,
- checklist for review or readiness,
- rubric for peer review or facilitator scoring,
- matrix for access controls, ownership, risks, or routing,
- roadmap for 30/60/90 plans,
- backlog for pilot implementation,
- spec template for functional or technical requirements,
- evidence record for validation and handoff,
- executive readout template for management decisions.

## Output Format

Provide an artifact register:

| Artifact | Type | Used In | Purpose | Required Sections | Format | Review Method | Slide Link |
|---|---|---|---|---|---|---|---|

Then provide:

- missing artifact warnings,
- template complexity notes,
- recommended creation order,
- handoff notes for document/spreadsheet/deck builders.

## Quality Bar

Every activity that asks learners to produce something should have a clear artifact or template.

Every artifact should answer:

- what learners put into it,
- what good output looks like,
- how it will be reviewed,
- how it connects to the final course outcome.

For management-facing programs, add an executive-readiness check:

- concise enough for management,
- decision-oriented,
- owner and next-action fields included,
- risks and assumptions visible,
- pilot-ready rather than classroom-only.

## Handoff

Hand the artifact register to:

1. `course-slide-planner` to ensure artifact slides are represented.
2. `course-slide-visual-planner` for canvas and activity visuals.
3. document/spreadsheet/presentation skills if actual templates must be created.
4. `course-slide` for embedding or linking templates in the final deck.
