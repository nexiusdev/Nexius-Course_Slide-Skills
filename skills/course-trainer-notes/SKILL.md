# Course Trainer Notes Skill

## Purpose

Create trainer-ready notes that let someone else deliver the course with
confidence. Notes should carry the detail that does not belong on the slide.

## Use When

Use this skill when asked to:

- add speaker notes,
- expand trainer notes,
- make a deck trainer-ready,
- create facilitation notes,
- prepare notes for a new trainer,
- improve PPT notes sections.

## Note Structure

For each teaching slide, include:

1. `Teaching purpose`: why this slide exists.
2. `How to explain`: concise trainer script or explanation.
3. `Example`: concrete workplace or course-specific example.
4. `Trainer prompt`: question to ask the room.
5. `Activity instruction`: only when the slide requires participant action.
6. `Timing cue`: optional but useful for lab and activity slides.
7. `Transition cue`: how to move to the next slide.

## Writing Rules

- Write for a trainer, not for the audience.
- Keep notes practical and specific.
- Include examples tied to the course domain.
- Put nuance in notes rather than on the slide.
- Avoid long lectures unless the slide is a key concept.
- For activity slides, specify timebox, output, and readout format.

## Workflow

1. Read the slide purpose and visible content.
2. Identify what the trainer must say that is not on the slide.
3. Add a concrete example.
4. Add a question to create engagement.
5. Add activity instructions where relevant.
6. Add a transition cue.
7. Check that notes match the slide and do not introduce unsupported content.

## Quality Checks

Confirm:

- every slide has notes,
- notes explain the concept plainly,
- new trainers have enough guidance,
- activity slides include output and timebox,
- transitions maintain teaching flow,
- notes do not duplicate slide text without adding value.
