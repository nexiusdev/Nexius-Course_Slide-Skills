# Course Learning Experience Designer

## Purpose

Use this skill before slide-count planning for substantial courses. Its job is to make the course teach well before it becomes slides.

Do not build the PowerPoint here. Produce a learning experience blueprint that slide planning, visual planning, artifact planning, and deck building can follow.

## Required Inputs

Start from the course brief, management approval document, existing outline, learning objectives, agenda, target audience, duration, delivery mode, and expected outputs.

If the course requirements are unclear, inspect the real source artifact first and state assumptions explicitly.

## Learning Design Workflow

1. Identify the learner profile:
   - job roles,
   - prior knowledge,
   - confidence level,
   - technical maturity,
   - likely anxiety or misconceptions,
   - real workplace context.
2. Define behavior-change outcomes:
   - what learners should understand,
   - what they should be able to do,
   - what they should produce,
   - what decisions they should make differently after the course.
3. Build the learning arc:
   - start with relevance and motivation,
   - introduce concepts in dependency order,
   - show worked examples before asking learners to build,
   - alternate concept, example, practice, debrief, and takeaway,
   - end with transfer to real work.
4. Design mastery checks:
   - quick polls,
   - scenario questions,
   - reflection prompts,
   - peer review rubrics,
   - artifact acceptance criteria,
   - capstone readout criteria.
5. Design activities:
   - activity purpose,
   - instructions,
   - timebox,
   - team roles,
   - expected output,
   - sample answer or model response,
   - debrief method,
   - fallback if learners get stuck.
6. Check cognitive load:
   - avoid too many heavy concepts in a row,
   - add examples before difficult activities,
   - insert debriefs after labs,
   - split overloaded modules,
   - create recovery moments after dense technical content.
7. Define trainer facilitation needs:
   - talk-track purpose,
   - questions to ask,
   - transitions,
   - common misconceptions,
   - energy resets,
   - timing cues,
   - debrief prompts.

## Output Format

Provide:

- audience and assumptions,
- behavior-change outcomes,
- day/module learning journey,
- concept dependency order,
- activity and assessment map,
- facilitation notes by module,
- cognitive-load risks,
- recommendations for slide planning.

For multi-day courses, use a table:

| Day/Module | Learner Goal | Concept Sequence | Practice Activity | Mastery Check | Facilitation Notes | Cognitive Load Risk |
|---|---|---|---|---|---|---|

## Quality Bar

A strong learning experience blueprint should make it clear:

- why learners care,
- what they can do afterward,
- how the course progresses from simple to complex,
- where learners practice,
- where learning is checked,
- how outputs become usable workplace artifacts,
- what the trainer must do beyond reading slides.
