# Course Program Design Skill

## Purpose

Create a practical course program that connects business outcomes, learner
capability, daily teaching flow, hands-on activities, and final deliverables.
The result should be useful to sponsors, trainers, and slide builders.

## Use When

Use this skill when asked to create or revise:

- a course outline,
- a detailed course program,
- a multi-day training curriculum,
- a concept-and-lab learning sequence,
- a project-led training plan,
- a course architecture for management review.

## Inputs

Gather:

- target audience and learner roles,
- business problem or transformation objective,
- course length and delivery format,
- required concepts,
- required tools or platforms,
- hands-on outputs,
- governance or risk constraints,
- final management deliverables.

If information is incomplete, state assumptions and continue unless the missing
detail changes audience, course duration, approval scope, or tool policy.

## Workflow

1. Start with the business outcome, not the topic list.
2. Define what participants must be able to do after the course.
3. Separate concept foundation from hands-on production work.
4. Convert each course day or module into:
   - purpose,
   - core concepts,
   - activities,
   - participant outputs,
   - trainer emphasis,
   - management value.
5. Include governance, review, and risk controls as course content, not as an
   afterthought.
6. End with a practical implementation path or roadmap.

## Recommended Course Pattern

For transformation courses, prefer:

- Days 1-2: shared language, operating model, governance, tool discipline, and
  conceptual foundation.
- Days 3-5: hands-on labs that produce real artifacts.
- Final session: executive readout, ownership model, metrics, and next actions.

## Output Structure

Use this structure unless the user asks otherwise:

1. Program summary.
2. Program outcomes.
3. Course structure table.
4. Detailed day-by-day or module-by-module program.
5. Participant outputs.
6. Trainer support requirements.
7. Governance and risk controls.
8. Final deliverables.
9. Post-course implementation path.
10. Open assumptions or decisions.

## Quality Checks

Before finalizing, confirm:

- the course is tied to a real business outcome,
- each day has a clear job,
- activities produce reusable artifacts,
- trainers know what to emphasize,
- governance is visible,
- the course avoids generic awareness training,
- final outputs can be used after the class.
