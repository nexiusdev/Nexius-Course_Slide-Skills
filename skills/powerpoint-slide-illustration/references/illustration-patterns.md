# Professional Slide Illustration Patterns

Use this reference when planning or generating PowerPoint teaching visuals.

## Planning Table Columns

For each slide, create a table with:

| Column | Purpose |
| --- | --- |
| Slide | Number and title |
| Teaching intent | What the trainer must explain |
| Visual job | What the audience should understand by looking |
| Visual type | Hub, loop, bridge, comparison, stack, gate, trail, canvas |
| Asset route | Source, adapt, generate, or draw |
| Labels | Which labels stay editable in PowerPoint |
| Notes purpose | How speaker notes should guide the trainer |

## Pattern Library

### Hub-and-Spoke System

Use for central intelligence, platform, agent, operating system, or coordination
models. Put the main concept in the center and 4-6 capabilities around it.
Good for "six parts of an agent", "AI operating system", "frontier firm hub",
and "model router".

### Circular Loop

Use for repeatable behaviour: observe-think-act-reflect, brief-delegate-review,
monitor-escalate-improve, feedback learning loops. Keep loop labels short.

### Workflow Bridge

Use when one artifact becomes another: workflow map -> boundaries -> agent,
manual process -> AI-assisted workflow, prompt -> context -> output.

### Before/After Transformation

Use for strategic shifts. Make the left side visibly fragmented or manual and
the right side visibly coordinated or governed.

### Layered Stack

Use for architecture, governance, context engineering, or capability layers.
Each layer should have a short label and a visual distinction.

### Decision Gate

Use for "when should we do this?" slides. Show criteria as gates or checkpoints,
ending in proceed / redesign / stop.

### Permission Model

Use for allowed / must ask / never, risk zones, approval paths, and tool access.
Use green/amber/red consistently.

### Evidence Trail

Use for auditability and governance. Show inputs, sources, decisions,
exceptions, outputs, and reviewer.

### Canvas Preview

Use for exercises. Show the worksheet/canvas as a polished preview so learners
understand the artifact they are producing.

## Professional Look Criteria

Professional illustration should have at least four of these:

- intentional composition and focal point,
- coherent icon family,
- restrained color palette,
- visual hierarchy,
- depth, glow, shadow, or texture where appropriate,
- meaningful relationships shown by paths, loops, or grouping,
- enough negative space for slide title and callout.

Avoid:

- random clipart,
- tiny icons scattered without structure,
- unlicensed marketplace previews,
- plain rectangles pretending to be illustrations,
- labels too small to read in presentation mode.
